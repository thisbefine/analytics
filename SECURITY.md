# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this package, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please email us at: **security@thisbefine.com**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

## Response Timeline

- We will acknowledge your report within 48 hours
- We will provide a detailed response within 7 days
- We will work with you to understand and resolve the issue

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Security Best Practices

When using this SDK:

1. **Keep your API key private** - Never commit API keys to public repositories
2. **Use environment variables** - Store API keys in `NEXT_PUBLIC_*` or similar
3. **Review tracked data** - Ensure you're not accidentally tracking sensitive user data
4. **Keep updated** - Use the latest version for security patches
